# Video to Docs

将视频转换为带截图和文字说明的专业操作教程文档。

## 工作流程

### 1. 确认输入与署名

首先确认用户提供的视频文件路径。如果路径不存在，提示用户提供正确路径。

同时询问用户文档署名偏好（使用 AskUserQuestion 工具）：

- **署名方式**：询问用户希望文档页脚显示什么署名信息。选项示例：
  - `本文档由 [用户名/公司名] 编写`
  - `本文档由 AI 自动生成`
  - 不添加署名（留空）
- 如果用户未指定，默认使用 `本文档由 AI 从录屏自动生成`

### 2. 提取关键帧

调用脚本提取视频关键帧：

```bash
~/.claude/skills/video-to-docs/scripts/convert.sh <video_path> [output_dir]
```

脚本会自动：
- 检查并安装依赖（ffmpeg、pandoc）
- 使用 scene detection 提取关键帧
- 如果没有场景变化，按固定间隔提取
- 输出图片到 `docs/images/` 目录

### 3. AI 分析图片（两阶段）

#### 前置：检查转录文件

在开始分析前，检查输出目录中是否存在以下文件：

- `frame_timestamps.json`：帧时间戳文件（由 convert.sh 自动生成）
- `transcript.json`：语音转录文件（如果视频包含语音且已安装 whisper-cpp）

如果 `transcript.json` 存在且非空，说明视频有语音解说，后续分析应充分利用转录内容。

#### 阶段 A：逐帧分析

逐个读取提取的图片，对每张截图进行结构化分析。

**时间戳对齐**（当 transcript.json 存在时）：

对于每帧，根据 `frame_timestamps.json` 中的时间戳，从 `transcript.json` 中匹配关联的转录段落：
- 匹配规则：找到所有 `|segment_center - frame_time| ≤ 5秒` 的转录段落
- 如果无匹配，取最近的 1-2 个段落作为上下文
- 合并匹配段落的文本作为该帧的「语音上下文」

对于每张图片：
1. 使用 Read 工具读取图片（返回 CDN URL）
2. 使用 `mcp__4_5v_mcp__analyze_image` 分析，提示词为：

```
分析这张软件操作截图，请提供以下信息：

1. **操作名称**（6字以内的动词短语，如"上传图片"、"选择模板"）
2. **操作描述**（用祈使句描述用户正在做什么，UI元素用**粗体**标注，如"点击**上传**按钮，在弹出窗口中选择图片文件"）
3. **界面要素**（截图中的重要 UI 元素：按钮名称、菜单路径、输入框标签等）
4. **建议提示类型**（从以下选择一个，如无需提示则填"无"：NOTE/TIP/WARNING/IMPORTANT。生成的文档中用 `> **💡 TIP:**` 等加粗前缀格式，不用 `[!TYPE]` 语法）
5. **建议提示内容**（一句话的实用建议或注意事项）
```

**如果有语音上下文**，在提示词末尾追加：

```
附加信息：以下是该操作步骤对应的语音解说内容：
"[匹配的转录文本]"

请结合截图和语音解说内容进行分析。如果语音中提到了截图看不到的信息（如快捷键、操作意图、注意事项），请优先采用语音内容。
```

3. 保存每帧的分析结果，用于后续文档生成

#### 阶段 B：全局上下文

所有帧分析完成后，综合全部信息确定：

1. **教程标题**：简洁明了，体现工具名 + 操作目标
2. **概述段落**：2-3 句话说明本教程做什么、适合谁
3. **前置条件**：列出使用前需要的准备工作（仅列出从截图中能合理推断的）
4. **验证方法**：如何确认操作成功完成
5. **常见问题**：预估 2-3 个可能遇到的问题和解决方案
6. **总结要点**：3 个关键要点

**如果存在 transcript.json**，在阶段 B 分析时：
- 将完整转录文本作为额外参考，用于更准确地确定教程标题（标题可能直接来自语音）
- 从转录中提取前置条件（语音可能提到准备工作、版本要求等）
- 从转录中提取注意事项作为常见问题的补充来源

### 4. 生成文档

根据阶段 A 和阶段 B 的分析结果，生成专业格式的 Markdown 文档。

#### 格式规则

1. **标题层级**：文档只用一个 `#`（H1，教程标题）。章节用 `##`（H2），步骤用 `###`（H3）
2. **步骤描述**：使用祈使句（"点击...""选择..."）。UI 元素用 **粗体**。菜单路径用 **菜单** > **子菜单** 连接
3. **图片**：放在操作描述之后。alt 属性格式为 `步骤 N - 操作名称 | 简短说明`
4. **Callout**：使用 `> **💡 TIP:**` / `> **📌 NOTE:**` / `> **⚠️ WARNING:**` / `> **🔔 IMPORTANT:**` 格式，只在有实际信息价值时添加，不要每个步骤都加。md-to-pdf 不支持 `> [!TYPE]` 语法，必须用加粗前缀格式
5. **分隔线**：步骤之间用 `---` 分隔
6. **编号列表**：当一步包含多个子操作时，使用编号列表
7. **UI 路径**：菜单导航用 **菜单** > **子菜单** > **选项** 格式
8. **可选步骤**：标记为 `（可选）`


### 5. 生成 PDF

生成 Markdown 文件后，转换为 PDF。

**优先方案**：使用 md-to-pdf

由于 Markdown 文件头部已包含 `pdf_options` 和 `stylesheet` 配置，直接执行：

```bash
npx -y md-to-pdf <output.md>
```

**备选方案**：使用 pandoc + xelatex（更好的中文排版）

如果 md-to-pdf 效果不理想，使用 pandoc。需要将 Markdown 头部的 YAML front matter 替换为 pandoc 格式：

```yaml
---
title: "[教程标题]"
lang: zh-CN
papersize: a4
fontsize: 11pt
CJKmainfont: "PingFang SC"
mainfont: "PingFang SC"
geometry: "margin=2cm"
header-includes:
  - \usepackage{longtable}
  - \usepackage{booktabs}
  - \usepackage{xcolor}
  - \definecolor{linkblue}{HTML}{2563EB}
  - \usepackage[colorlinks=true,linkcolor=linkblue,urlcolor=linkblue]{hyperref}
---
```

然后执行：

```bash
pandoc <output.md> -o <output.pdf> --pdf-engine=xelatex
```

## 使用示例

**用户：** 把这个录屏转成教程 `/Users/paul/Downloads/demo.mov`

**响应：**
1. 确认视频路径，询问署名偏好
2. 调用 convert.sh 提取关键帧（同时提取帧时间戳和语音转录）
3. 检查 transcript.json 是否存在（判断是否有语音解说）
4. 阶段 A：逐帧分析图片内容（如有转录，匹配关联语音作为上下文）
5. 阶段 B：生成全局上下文（标题、概述、前置条件等，参考完整转录）
6. 按模板生成 Markdown 文档
7. 转换为 PDF
8. 告知输出位置

## 配置选项

可通过环境变量调整：

| 变量 | 说明 | 默认值 |
|------|------|--------|
| `SCENE_THRESHOLD` | 场景检测灵敏度 (0-1) | 0.1 |
| `MAX_FRAMES` | 最大帧数 | 30 |
| `WHISPER_MODEL` | Whisper 模型文件路径 | `~/.claude/skills/video-to-docs/models/ggml-large-v3-q5_0.bin` |
| `WHISPER_LANGUAGE` | 转录语言 | `zh` |
| `ENABLE_TRANSCRIPTION` | 是否启用转录（`auto`/`yes`/`no`） | `auto` |

## 注意事项

1. **视频长度限制**：建议 5 分钟以内，过长会导致处理缓慢
2. **隐私保护**：视频、截图和音频仅在本地处理，转录使用本地 whisper.cpp
3. **敏感信息**：提醒用户检查输出文档是否包含敏感信息后再分享
4. **AI 幻觉**：阶段 B 的全局分析基于截图中可见内容推断，不要编造不存在的信息
5. **语音转录可选**：未安装 whisper-cpp 时流程正常运行，仅跳过转录步骤。安装方式：`brew install whisper-cpp`
6. **AI 分析失败处理**：如果某帧的 vision API 调用失败（超时、报错等），重试一次。如仍失败，生成占位描述：「步骤 N：[未能自动识别，请手动补充]」，继续处理后续帧，最终报告哪些帧分析失败
